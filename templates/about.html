Project Executive Summary

This web application predicts match outcomes and expected scores across three distinct sports using Scikit-Learn's Random Forest algorithms. Unlike basic predictors that simply forecast a winner, this app employs multi-output regression pipelines to predict expected scores (goals, runs, points) along with **probabilistic classifiers** to gauge the exact confidence percentage for each team.

The code is designed to be fully self-contained. It features dynamic dataset generation, feature preprocessing, model evaluation, and a modern asynchronous Flask API.

4. Split & Train

5. Inference

Presentation Q&A (Viva-Voce Guide)

Prepare for your college project evaluation by reviewing these common machine learning questions.

Q: Why did you choose Random Forest over simple Logistic/Linear Regression?

A: Random Forest is an ensemble learning method based on decision trees. Unlike linear models, it handles complex, non-linear relationships and high-dimensional categorical features (like team vs team on specific venues) without assuming a linear boundary, resulting in higher predictive accuracy and robustness to outliers.

Q: How do you handle categorical columns like team names in the machine learning code?

A: We use Scikit-Learn's `OneHotEncoder`. This preprocessor converts categorical strings (e.g., "Mumbai Indians") into binary columns (1s and 0s) so the mathematical algorithms can process them. We bundle the `OneHotEncoder` and the model into a Scikit-Learn `Pipeline` to ensure preprocessing is applied automatically during both training and prediction.

Q: What is the benefit of predicting win probability alongside the winning team?

A: Simple winner classification only gives a label (e.g., Team A). By using the `predict_proba()` method, we extract the model's confidence distribution. This allows us to display the prediction as a percentage (e.g., 68.5% win chance), representing the probability of occurrence, which is a much more useful and professional metric in sports analytics.
